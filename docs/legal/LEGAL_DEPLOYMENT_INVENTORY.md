# Legal deployment inventory

This inventory classifies the current legal documents and work products for the VANTAM website. It separates what can be public now, what depends on a dedicated implementation, what belongs in the client pack before acceptance, and what stays internal.

## Public website

- **General Terms**: Public because the website must show the contractual reference terms before acceptance and the public identity block belongs with that legal surface.
- **Withdrawal Information**: Public because consumers need the standard withdrawal explanation before any distance-service contract is accepted.
- **Privacy Policy**: Public because the website collects enquiry data and must explain how it is used, stored, and shared.

## Public only after its dedicated implementation

- **Online Withdrawal Function**: Public only after the withdrawal workflow is implemented and verified, because the page must not falsely claim that the function is already live.

## Client-specific and supplied before acceptance

- **Consultation Confirmation**: Client-specific because it records the exact consultation scope, timing, price, and acceptance details for one engagement.
- **Service Offer / Appendix**: Client-specific because it sets the agreed scope, price, dependencies, and exclusions for one client or matter.
- **Model Withdrawal Form**: Client-specific when supplied in the contract pack because it supports a specific consumer contract context and should travel with the acceptance pack.
- **Early-Start Request**: Client-specific because it only applies when a consumer explicitly asks for work to start during the withdrawal period.
- **Final accepted contract pack**: Client-specific because it contains the accepted versions, contract evidence, and the complete trader address supplied before acceptance.

## Internal only

- **Acceptance Record**: Internal evidence only because it stores proof of versions, timestamps, and acceptance method.
- **Invoice Wording Guide**: Internal only because it controls billing language and should not be exposed as public legal content.
- **Legal Source Register**: Internal only because it records source material, not customer-facing terms.
- **Open Blockers**: Internal only because it tracks launch blockers and unresolved legal or operational items.
- **Operational workflows and checklists**: Internal only because they describe staff process, evidence capture, and retention handling.
